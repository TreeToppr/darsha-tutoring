# DarshaTutor Roadmap

This roadmap keeps DarshaTutor focused. It is not a full wishlist.

## Product principle

DarshaTutor should help tutoring relationships run smoothly and preserve educational continuity.

The platform should reduce reliance on scattered messages, screenshots, memory, spreadsheets, and manual follow-up.

## v2.0.0 - Tutoring Operations Platform

Status: Released

Purpose:

Create the operational base for tutoring.

Core capabilities:

- User sign-up and sign-in
- Parent, student, tutor, and admin areas
- Lesson booking and management
- Lesson status tracking
- Payment/invoice workflow
- Notifications
- Public homepage

## v3.0.0 - Educational Continuity Platform

Status: Active

Purpose:

Preserve the student’s learning context.

Success criteria:

A tutor can open a student profile and understand:

- student details
- current subjects and year level
- goals
- concerns
- skill gaps
- progress history
- recent lesson activity
- next recommended actions

Likely feature areas:

- student profiles
- goals
- concerns
- skill observations
- learning timeline
- parent-visible progress view
- tutor-facing student overview
- basic evidence/history capture

Definition of done:

- A tutor can use DarshaTutor instead of scattered notes to understand a student.
- A parent can see meaningful progress information.
- The data model supports future lesson rooms and AI summaries.
- v3.0.0 issues are either closed, moved, or deliberately deferred.

## v3.1.0 - Lesson Workspace & Lesson Rooms

Status: Planned

Purpose:

Make each lesson a working space.

Likely feature areas:

- lesson room page
- lesson preparation notes
- lesson notes
- resources
- homework
- lesson summary
- follow-up actions
- transcript/recording integration if appropriate
- parent/student lesson recap

Dependency:

Requires v3.0.0 student history foundation.

## v3.2.0 - Educational Intelligence

Status: Planned

Purpose:

Use AI and educational memory to support tutors, parents, and students.

Likely feature areas:

- AI lesson summary
- AI student overview
- pattern detection
- suggested next steps
- tutor preparation assistant
- parent-friendly progress explanation
- student study support

Dependency:

Requires reliable educational data from v3.0.0 and lesson data from v3.1.0.

## v3.2.0 - Parent & Tutor Operations

Status: Planned

Purpose:

Help parents and tutors manage the tutoring relationship inside DarshaTutor.

Likely feature areas:

- lesson requests
- reschedule workflow
- cancellation workflow
- notifications
- contact tutor workflow
- parent activity feed
- communication preferences
- announcements
- help centre
- AI support assistant
- smart actions

Risk:

This milestone is useful, but it can easily become too broad. Prioritise only the workflows that reduce real operational pain.

## Platform Infrastructure & Reliability

Status: Ongoing

Purpose:

Make the platform stable, secure, maintainable, and trustworthy.

Feature areas:

- error handling
- audit logging
- security review
- environment documentation
- deployment stability
- database migration discipline
- testing
- backup/recovery thinking
- performance
- monitoring

## Backlog / Future Ideas

Use this for ideas that are interesting but not current priorities.

Rules:

- Capture the idea.
- Do not build it immediately.
- Review periodically.
- Promote only when it supports the current product direction.
